ICommunication Methods

The ICommunication type exposes the following members.

Methods
  NameDescription
AddConnection
Add new connection to communication process Return true on success, false if connection was not added to communication
DisableSendingData
Disable sending data with specific mimeType to connection specified by connectionId
GetConnectionIds
Get all connection IDs participating in this communication Return array of string with all connection IDs participating in this communication
GetConnectionIdsSupportMimeType
Get all connections IDs which has enabled specified mimeType Return array of string with connection IDs which support specified mimeType
GetConnectionStreamIds
Get all generated stream IDs associated with one specified connection Return array of IDs associated with specified connection. If there is no generated ID associated with connection, then return empty array
GetMediaStreamHeader
Get media header (first bytes of video/audio, eg. Initialization segment) for specified streamId Return First bytes of stream or empty byte array if header is not stored
GetNewUniqueStreamId
Return unique id for new stream. Range of ids is > 50000.
IsCommunicationAllowed
Check constraints if communication is allowed (check is done based on Master an Slave constraints) Return true if communication is allowed, otherwise return false
IsCommunicationFull
Check if connection is full Return true if connection is full, otherwise return false
IsMasterConnected
Check if master is still connected to server Return true if master is connected to server, otherwise return false
IsMediaStreamHeaderExist
Check if media stream header exist Return true if exist, otherwise false
IsMimeTypeEnabled
Check if specified mimeType is enabled for specified connection (by default all mimeTypes should be enabled) Return true if mime type is enabled, otherwise return false
IsRequestStreamIdAllowed
Check if requestStreamId action is allowed by specific connection Return true if requestStreamId is allowed, otherwise return false
RemoveConnection
Remove connection from communication process
SetMasterConstraint
Set constraints for master connection (has higher priority than slave constraints, [for example if master has disabled video then slaves also has disabled video])
SetMediaStreamHeader
Set media header for specific streamId
SetSlaveConstraint
Set constraints for slave connection (has lower priory than master constraints, [for example if slave has disabled video, master could have video enabled])
StopStream
Stop stream. Check if stream is associated with specific connection. On success release resources associated with stream. Return true on success, otherwise return false.
